API REST

Définitions

API

Une API (Application Programming Interface) est un ensemble de fonctionnalités et de règles existant dans un logiciel permettant d’interagir avec celui-ci de manière automatisée (plutôt que de passer par une interface utilisateur). L’API peut être vue comme un contrat passé entre le logiciel qui la propose et d’autres entités, telles que des logiciels ou matériels tiers.

Dans une classe ou un objet JavaScript, les méthodes et les propriétés publiques peuvent être considérées comme une API, car elles exposent des fonctionnalités qui peuvent être utilisées par d’autres parties de l’application, ou importées dans d’autres programmes.

Dans le cas d’un service web, une API est un ensemble de règles et de spécifications qui permettent à des logiciels de communiquer entre eux.

Les API web permettent à des applications de communiquer entre elles, indépendamment de leur langage de programmation.

API n’est pas une technologie, mais un concept. Il existe de nombreux types d’API, tels que :

  • les API de base de données
  • les API de système d’exploitation
  • les API de bibliothèques de code
  • les API JavaScript (Fetch, History, Web Storage, etc.)
  • les API de services web

Lorsque l’on parle d’API REST, on fait généralement référence à des services web qui permettent à des systèmes informatiques d’échanger des données via le protocole HTTP.

REST

REST (REpresentational State Transfer) est un style d’architecture pour la conception de services web. Les API REST utilisent les principes fondamentaux du protocole HTTP (Hypertext Transfer Protocol), qui est le protocole de communication de base d’Internet.

Caractéristiques clés d’une API REST

Ressources

Les données ou services exposés par une API REST sont représentés comme des ressources. Chaque ressource est identifiée de manière unique par une URI (Uniform Resource Identifier).

Opérations standardisées

Les actions sur les ressources sont effectuées en utilisant les méthodes HTTP standard, telles que :

  • GET pour récupérer des données,
  • POST pour créer de nouvelles ressources,
  • PUT pour mettre à jour des ressources existantes,
  • DELETE pour supprimer des ressources.

Stateless (sans état)

Chaque requête du client au serveur contient toutes les informations nécessaires pour comprendre et traiter la demande. Le serveur ne conserve pas d’informations sur l’état du client entre les requêtes.

Représentation des ressources

Les données échangées entre le client et le serveur sont généralement au format JSON (JavaScript Object Notation) ou XML (eXtensible Markup Language). Ces représentations décrivent l’état de la ressource à un moment donné.

L’utilisation de ces principes permet aux API REST d’être simples, évolutives, et facilement compréhensibles. Elles sont largement utilisées pour le développement d’applications web et mobiles, ainsi que pour l’intégration de services et la création d’architectures distribuées.

Notes

Le terme API REST est fréquemment utilisé pour désigner des API web qui respectent partiellement ou superficiellement les principes de REST, mais il est important de noter que de nombreuses API qui se disent REST ne respectent pas tous les principes de REST.

Alternatives à REST

Il existe d’autres styles d’architecture pour les services web, tels que SOAP (Simple Object Access Protocol), qui est un protocole basé sur XML, et GraphQL, qui est une spécification de requêtes et de manipulation de données.
